4. HTTP
"HTTP/1.1"은 HTTP의 버전, "200 OK"는 아래 응답 코드와 메시지의 조합, 헤더 부분과 바디 부분은
서버 정보와 요청받은 시각 등, 응답 본문이 함께 전송된다. 일반적인 웹 문서를 요청했을 경우에는
HTML로 시작하는 웹문서가 전송된다.

4-1) 응답 코드는 보통 아래와 같이 분류된다.
- 1XX : 정보 전달, 작업을 진행하고 있음을 의미
- 2XX : 성공, 작업을 성공적으로 처리했음을 의미
- 3XX : 리다이렉션, 요청 완료를 위해 리다이렉션이 이루어져야 한다는 의미
- 4XX : 클라이언트 오류, 클라이언트에서 요청한 내용이 올바르지 않다는 의미
- 5XX : 서버 오류, 서버에서 요청을 처리할 수 없었음을 의미
- 6XX 이후 : 정의되지 않음, 사용자 정의 영역에 해당한다.

4-2) 지원하는 요청 메시지는 다음과 같다.
- GET : 클라이언트가 서버에게 URL에 해당하는 자료의 전송을 요청한다.
- POST : 클라이언트가 서버에서 처리할 수 있는 자료를 보낸다. 예로 게시판에 글을 쓸 때 클라이언트의 문서가 서버로 전송되어야 한다. 멱등성을 보장하지 않는다.
- HEAD : GET 요청으로 반환될 데이터 중에 헤더 부분에 해당하는 데이터만 요청한다.
- PATCH : 클라이언트가 서버에게 지정한 URL의 데이터를 부분적으로 수정할 것을 요청한다.
- PUT : 클라이언트가 서버에게 지정한 URL에 지정한 데이터를 저장할 것을 요청한다.
- DELETE : 클라이언트가 서버에게 지정한 URL 정보를 제거할 것을 요청한다.
- TRACE : 클라이언트가 서버에게 송신한 요청의 내용을 반환해 줄 것을 요청한다.
- CONNECT : 클라이언트가 특정 종류의 프록시 서버에게 연결을 요청한다.
- OPTIONS : 해당 URL에서 지원하는 요청 메시지의 목록을 요청한다.

    ///// 1-4) 구조와 핵심 태그 소개

    문장이 모이면 페이지가 된다. 페이지가 모이면 책이 된다. 이제 구조를 만드는 태그를 볼 것임
    ( 책 > 표지 >제목, 저자 )

    - title : 웹 페이지 타이틀을 정의한다.
    - meta charset="utf-8" | 태그+속성이지만, 중요한 것이라 넣음( 전세계 문자열 공통 호환 -> 한글로 해도 깨지지 않는다 )
    = 브라우저에게 UTF-8로 문서를 읽으라고 얘기하는 것( meta 속성 중 이것 말고는 몰라도 된다 )

    - : HTML이 담겨있다는 뜻 ( html보다 상단에 위치하는 HTML 최상위 태그 )
    - html : head와 body를 묶는 태그 ( html - head - body | 부모 자식 관계 )
    - head : 본문을 설명하는 태그 body를 묶는다( body와 같이 구분자로서 JS의 어떤 재료를 쓰겠다고 미리 선언해주는 역할 등 )
    - body : 본문을 이 태그로 묶겠다는 뜻( 사람들과의 약속이다 )

    이 모든 것들은 한 페이지를 구성하는 구조로 이 모든 것들 위에 존재하는 것이 '링크'이다. 태그로 이것이 없었다면
    검색엔진은 없었을 것임. a 태그가 있어서 웹 페이지에서 다른 웹 페이지로 이동할 수 있게 된 것

    - a : 어떤 링크에 닷을 내릴 건지 추가 설명해줄 '속성'이 필요하다. ( h(하이퍼링크) + ref(참고) )
    ( ex. a href="https://html.spec.whatwg.org/multipage/">Hypertext Markup Language (HTML)/a )
    + 추가 속성
    target="_blank" : 새 탭에서 열겠다는 뜻 /
    title="HTML5 specification" : 커서 위치 시에 툴팁을 표시하겠다는 뜻 /

    ///// 1-7) 서버와 클라이언트
    클라이언트 컴퓨터 - 서버 컴퓨터 두 개로 나뉘어져있다. 웹 브라우저가 설치된 웹 클라이언트가 웹 서버로 파일 요청을 보내면
    전기적 신호가 웹 서버로 전달되고 웹 서버는 해당 파일 내용을 코드로 보내준다. 통신이 일어나고 웹 브라우저가 설치된
    웹 클라이언트 컴퓨터는 코드를 해석하여 사용자에게 보여주게 된다.
    웹을 사용하기 위해 웹 클라이언트, 웹 서버라는 말로 쓸 수도 있고
    게임을 하기 위해서 게임 클라이언트, 게임 서버라고 부를 수도 있다. 경우에 따라 다르게 부를 수도 있는 셈.
    내 컴퓨터에 웹 서버를 설치해서 내 문서를 전 세계 사람들이 볼 수 있게 할 수도 있고 외부 특정 서버가 있는
    대행업체에 일을 맡길 수도 있다( 웹 호스팅 )
